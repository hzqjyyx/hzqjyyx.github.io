Layered Architecture

pros: clean, modular, easy to update one component.
cons: duplicated work (error recovery), information may across layers

Two Internet hosts connected via two routers and the corresponding layers used at each hop. The application on each host executes read and write operations as if the processes were directly connected to each other by some kind of data pipe. Every other detail of the communication is hidden from each process. The underlying mechanisms that transmit data between the host computers are located in the lower protocol layers.

Application Layer

The application layer is where network application and their application-layer protocol reside.
HTTP, FTP, SMTP, DNS
Message

Transport Layer

It transports application-layer messages between application endpoints. TCP provides connection-oriented service, including guaranteed delivery, flow control, conguestion control. UDP is barebone of ip packets.
TCP,UDP
Segment

Network Layer

It is responsible for moving network-layer packets as datagrams from one host to another.
IP
Datagram

Link Layer

It helps to move a packet from one node (host or route) to next node.
WiFi, Ethernet, PPP
Frame

Physical Layer

Moves individual bits.

UDP and TCP

tcp and udp protocol do not contain ip address of host and destination. They have alreay existed in IP datagram.

UDP

The User Datagram Protocol (UDP) is a very thin protocol build on top of the Internet Protocol. The basic unit of data is a User datagram and the UDP protocol provides the same unreliable, connectionless service transferring user datagrams as the IP protocol does transferring its datagrams. The main difference is that the UDP protocol is an end-to-end protocol. That is, it contains enough information to transfer a user datagram from one process on the transmitting host to another process on the receiving host.

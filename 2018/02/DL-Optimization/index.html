Backprop property

Backprop low-variance

In classification problems, the classification error is a non-differentiable function of weights, the divergence function minimized is only a proxy (approximation) for classification error. Minimizing divergence may not minimize classification error.

Backprop fails to separate where perceptron succeeds

The perceptron may change greatly upon adding just a single new training instance

  • But it fits the training data well. The perceptron rule has low bias
    • Makes no errors if possible
  • But high variance
    • Swings wildly in response to small changes to input

Backprop is minimally changed by new training instances

  • Prefers consistency over perfection
  • It is a low-variance estimator, at the potential cost of bias.

In an MLP the lower layers “learn a representation” that enables linear separation by higher layers. Backpropagation will often not find a separating solution even though the solution is within the class of functions learnable by the network. This is because the separating solution is not a feasible optimum for the loss function.

Gradient descent can miss obvious answers and this may be a good thing.

Convergence

Definition: if the value updates arrive at a fixed point where the gradient is 0 and further updates do not change the estimate.

Error Surface

Popular hypothesis:

  • In large networks, saddle points are far more common than local minima
    • Frequency exponential in network size
  • Most local minima are equivalent
    • And close to global minimum

Saddle point: A point where

  • The slope is zero

  • The surface increases in some directions, but decreases in others

    • Some of the Eigenvalues of the Hessian are positive; others are negative

  • Gradient descent algorithms often get “stuck” in saddle points

Step Size

Conventional vector update rules for gradient descent: update entire vector against direction of gradient. The same learning rate is applied to all components.

Note : Gradient is perpendicular to equal value contour .

Note: Optimal learning rate is different for the different coordinates, so some axes will explore while some are slow.

Therefore, the learning rate must be lower than twice the smallest optimal learning rate for any component. Otherwise the learning will diverge. This, however, makes the learning very slow. And will oscillate in all directions.

However, when we think about 2D inputs, we notice that, descents are uncoupled.

The optimum of each coordinate is not affected by the other coordinates. We could optimize each coordinate independently.

Solution: Normalize the objective to have identical eccentricity in all directions. Then all of them will have identical optimal learning rates.

Issues:

Inverse of Hessian matrix is really expensive to compute.
Hessian may not be positive semi-definite, the algorithm can diverge.

Decaying learning rate

For complex models such as neural networks, the loss function is often not convex, having η>2ηopt\eta > 2 \eta^{opt}η>2η​opt​​ can actually help escape local optima. However, always having η>2ηopt\eta > 2 \eta^{opt}η>2η​opt​​ will ensure that you never find a solution

A common approach:

  1. Train with a fixed learning rate η\etaη until loss (or performance on a held-out data set) stagnates.
  2. η:=αη\eta := \alpha \etaη:=αη where α<1\alpha < 1α<1, typically 0.1
  3. Return to step 1 and continue training from where we left off

Summary

Better convergence can be obtained through

  • Second order methods that normalize the variation across dimensions, like Newton’s method
  • Adaptive or decaying learning rates that can improve convergence
  • Methods like Rprop that decouple the dimensions can improve convergence
  • Momentum methods which emphasize directions of steady improvement and deemphasize unstable direction

#Derivative-inspired algorithm

RProp

At each time:

  • If the derivative at the current location recommends continuing in the same direction as before (has not changed sign from earlier)
    • increase the step, and continue in the same direction
  • If the derivate has changed sign (we’ve overshot a minimum)
    • return to the previous position
    • reduce the step and reverse direction

A remarkably simple first-order algorithm, that is frequently much more efficient than gradient descent.

Quick prop

Quickprop employs the Newton updates with two modifications.

First, it treats each dimension independently, which eliminates the need to compute and invert expensive Hessians.

Momentum

The problem we want to eliminate:

With dimension-independent learning rates, the solution will converge smoothly in some directions, but oscillate or diverge in others.

Proposal:

  • Keep track of oscillations
  • Emphasize steps in directions that converge smoothly
  • Shrink steps in directions that bounce around…

Maintain a running average of all past steps

  • In directions in which the convergence is smooth, the average will have a large value. In other word, get longer in directions where gradient stays in the same sign.
  • In directions in which the estimate swings, the positive and negative swings will cancel out in the average. In other word, become shorter in directions where the sign keeps flipping

Update with the running average, rather than the current gradient.

Nestorov’s Accelerated Gradient

A modified version of momentum.

Change the order of operations.

At any iteration:

  • First extend the previous step using running average gradient

  • Then compute the gradient step at the resultant position

  • Add the two to obtain the final step

ΔWk=βΔWk−1−η∇WErr(Wk−1+βΔWk−1)Wk=Wk−1+ΔWk\begin{aligned} \Delta W^k = \beta \Delta W^{k-1} - \eta \nabla_WErr(W^{k-1}+\beta \Delta W^{k-1})\\ W^k = W^{k-1}+\Delta W^k \end{aligned} ​ΔW​k​​=βΔW​k−1​​−η∇​W​​Err(W​k−1​​+βΔW​k−1​​)​W​k​​=W​k−1​​+ΔW​k​​​​

Converges much faster

Incremental updates

Conventional gradient descent tries to simultaneously adjust the function at all training points, which is less efficient. Also, we may not fit data well when some point is close to our function while some it much farer to our hypnosis.

For incremental update, Stochastic Gradient Descent:

We adjust the function at one training point at a time, keep adjustments small.

Problem:

  • Random select item, or we will get cyclic behavior.
  • Except in the case of a perfect fit, even an optimal overall fit will look incorrect to individual instances
    • Correcting the function for individual instances will lead to never-ending, non-convergent updates
    • We must shrink the learning rate with iterations to prevent this
    • Correction for individual instances with the eventual miniscule learning rates will not modify the function

Intersection of Two Arrays

Given two arrays, write a function to compute their intersection.

Example:
Given nums1 = [1, 2, 2, 1], nums2 = [2, 2], return [2].

Note:

  • Each element in the result must be unique.
  • The result can be in any order.

Answer 3ms

Tags: Sort, Three pass
Time: O(nlg⁡n)O(n\lg n)O(nlgn), Space: O(n)O(n)O(n)
Explanation: It is worst in asymptotic time in the second solution, but it is compact and without overhead of HashSet. Besides, the dataset is small enough, so it works well.
At first, we sort nums1 and nums2. Only when nums[i] = nums[j], they can be in the intersection set. And we can proof that if nums[i] < nums[j], there would be no chance that nums[0…i] can be equal to nums[j], the only chance would be nums[i+k] larger than nums[i].
Also, once we enter the equal case, it need to be skipped the equal nums, because the intersection set requires: Each element in the result must be unique.

Answer 4ms

Tag: HashSet, One pass
Time: O(n)O(n)O(n), Space: O(n)O(n)O(n)
Explanation:

  1. Construct a HashSet of nums1, it could be nums2. In my opinion, it should be the array that has less elements. Let’s say we have nums1.length = 10 and nums2.length = 10000. We can early stop if we find all characters are in intersection set.
  2. Let’s iterate nums2 and try to remove from HashSet. If remove successfully, it means there is same element both in nums1 and nums2. And it is in set only one time because we remove it.
  3. Here we use a simple trick. nums2[1…i] has been seen and will be never come back. Also, the intersection set’s length is always less than or equal to nums2[1…i]. So we can reuse nums2 without create a new array.

Questions:

  • What is the time and space complexity of HashSet.

Insertion Sort List

Sort a linked list using insertion sort.


A graphical example of insertion sort. The partial sorted list (black) initially contains only the first element in the list.
With each iteration one element (red) is removed from the input data and inserted in-place into the sorted list

Algorithm of Insertion Sort:

  1. Insertion sort iterates, consuming one input element each repetition, and growing a sorted output list.
  2. At each iteration, insertion sort removes one element from the input data, finds the location it belongs within the sorted list, and inserts it there.
  3. It repeats until no input elements remain.

BFS

As the name BFS suggests, you are required to traverse the graph breadthwise as follows:

  1. First move horizontally and visit all the nodes of the current layer
  2. Move to the next layer

A graph can contain cycles, which may bring you to the same node again while traversing the graph. To avoid processing of same node again, use a boolean array which marks the node after it is processed. While visiting the nodes in the layer of a graph, store them in a manner such that you can traverse the corresponding child nodes in a similar order.

Use a queue to store the node and mark it as ‘visited’ until all its neighbours (vertices that are directly connected to it) are marked. The queue follows the First In First Out (FIFO) queuing method, and therefore, the neigbors of the node will be visited in the order in which they were inserted in the node i.e. the node that was inserted first will be visited first, and so on.

Given a graph G=(V,E)G = (V,E)G=(V,E) and a distinguished source vertex s, breadth-ﬁrst search systematically explores the edges of G to discover every vertex that is reachable from s. It computes the distance (smallest number of edges) from s to each reachable vertex. It also create breadth-ﬁrst trees at the same time, defined using u.pi.

It expands the frontier between discovered and undiscovered vertices uniformly across the breadth of the frontier. That is, the algorithm discovers all vertices at distance kkk from s before discovering any vertices at distance k−1k -1k−1.

Deﬁne the shortest-path distance d(s,v)d(s,v)d(s,v) from sss to vvv as the minimum number of edges in any path from vertex sss to vertex vvv; if there is no path from sss to vvv, then d(s,v)=∞d(s,v)=\inftyd(s,v)=∞. And breadth-ﬁrst search correctly computes shortest-path distances.

DFS

Explanation

Depth-ﬁrst search explores edges out of the most recently discovered vertex vvv that still has unexplored edges leaving it. Once all of vvv's edges have been explored, the search backtracks to explore edges leaving the vertex from which vvv was discovered. The predecessor subgraph of a depth-ﬁrst search forms a depth-ﬁrst forest comprising several depth-ﬁrst trees. The edges in EπE_\piEπ​ are tree edges.

Each vertex is initially white, is grayed when it is discovered in the search, and is blackened when it is ﬁnished. Besides creating a depth-ﬁrst forest, depth-ﬁrst search also timestamps each vertex. Each vertex vvv has two timestamps: the ﬁrst timestamp v.dv.dv.d records when vvv is ﬁrst discovered (and grayed), and the second timestamp v.fv.fv.f records when the search ﬁnishes examining vvv's adjacency list (and blackens vvv). Vertex u is WHITE before time u.du.du.d, GRAY between time u.du.du.d and time u.fu.fu.f, and BLACK thereafter.

Detail

The graph GGG is whether directed or undirected.

The running time of DFS is Θ(V+E)\Theta(V+E)Θ(V+E).

Property

the structure of the depth-ﬁrst trees exactly mirrors the structure of recursive calls of DFS-VISIT

parenthesis structure: If we represent the discovery of vertex u with a left parenthesis ‘(u’ and represent its ﬁnishing by a right parenthesis ‘u)’, then the history of discoveries and ﬁnishing makes a well-formed expression in the sense that the parentheses are properly nested.

Classification of Edges

Gπ=(V,Eπ)G_\pi=(V,E_\pi)Gπ​=(V,Eπ​), where Eπ={(v.π,v):v∈Vandv.π≠Nil}E_\pi=\{(v.\pi,v): v\in V and v.\pi \neq Nil\}Eπ​={(v.π,v):v∈Vandv.π≠Nil}

We can deﬁne four edge types in terms of the depth-ﬁrst forest GπG_\piGπ​ produced by a depth-ﬁrst search on G:

  1. Tree edges are edges in the depth-ﬁrst forest GπG_\piGπ​. Edge (u,v)(u,v)(u,v) is a tree edge if vvv was ﬁrst discovered by exploring edge (u,v)(u,v)(u,v)
  2. Back edges are those edges (u,v)(u,v)(u,v) connecting a vertex uuu to an ancestor vvv in a depth-ﬁrst tree. We consider self-loops, which may occur in directed graphs, to be back edges.
  3. Forward edges are those nontree edges (u,v)(u,v)(u,v) connecting a vertex uuu to a descendant vvv in a depth-ﬁrst tree.
  4. Cross edges are all other edges. They can go between vertices in the same depth-ﬁrst tree, as long as one vertex is not an ancestor of the other, or they can go between vertices in different depth-ﬁrst trees.

For an undirected graph, we classify the edge as the ﬁrst type in the classiﬁcation list that applies. And, forward and cross edges never occur in a depth-ﬁrst search of an undirected graph.

The key idea is that when we ﬁrst explore an edge (u,v)(u,v)(u,v) the color of vertex vvv tells us something about the edge:

  1. WHITE indicates a tree edge,
  2. GRAY indicates a back edge, and
  3. BLACK indicates a forward or cross edge.

Topological sort

Sometimes it is called DAG. A topological sort is a linear ordering of all its vertices such that if G contains an edge (u,v)(u,v)(u,v), then uuu appears before vvv in the ordering. (If the graph contains a cycle, then no linear ordering is possible.)

We can perform a topological sort in time Θ(V+E)\Theta(V+E)Θ(V+E), since depth-ﬁrst search takes Θ(V+E)\Theta(V+E)Θ(V+E) time and it takes O(1)O(1)O(1) time to insert each of the ∣V∣|V|∣V∣ vertices onto the front of the linked list.

GGG and GTG^TGT have exactly the same strongly connected components: uuu and vvv are reachable from each other in GGG if and only if they are reachable from each other in GTG^TGT.

Here is the explanation of why it works:

  1. We shall see that by considering vertices in the second depth-ﬁrst search in decreasing order of the ﬁnishing times that were computed in the ﬁrst depth-ﬁrst search, we are, in essence, visiting the vertices of the component graph (each of which corresponds to a strongly connected component of G) in topologically sorted order.
  2. each edge in GTG^TGT that goes between different strongly connected components goes from a component with an earlier ﬁnishing time (in the ﬁrst depth-ﬁrst search) to a component with a later ﬁnishing time.
  3. In general, when the depth-ﬁrst search of GTG^TGT in line 3 visits any strongly connected component, any edges out of that component must be to components that the search already visited. Each depth-ﬁrst tree, therefore, will be exactly one strongly connected component.

Shortest Path Problems

In shortest-path problem, we are given a weighted, directed graph G=(V,E)G=(V,E)G=(V,E), with weighted function www. We define shortest-path weight δ(u,v)\delta(u,v)δ(u,v) from uuu to vvv:

A shortest path from vertex uuu to vertex vvv is then deﬁned as any path ppp with weight w(p)=δ(u,v)w(p)=\delta(u,v)w(p)=δ(u,v).

Variant

  • Single-source shortest-paths problem: given a graph GGG, we want to ﬁnd a shortest path from a given source vertex sss to each vertex vvv.
  • Single-destination shortest-paths problem: Find a shortest path to a given destination vertex t from each vertex v. By reversing the direction of each edge in the graph, we can reduce this problem to a single-source problem.
  • Single-pair shortest-path problem: Find a shortest path from uuu to vvv for given vertices uuu to vvv. If we solve the single-source problem with source vertex u, we solve this problem also. Moreover, all known algorithms for this problem have the same worst-case asymptotic running time as the best single-source algorithms.
  • All-pairs shortest-paths problem: Find a shortest path from uuu to vvv for every pair of vertices uuu and vvv. Although we can solve this problem by running a single-source algorithm once from each vertex, we usually can solve it faster.

The Bellman-Ford algorithm

The Bellman-Ford algorithm solves the single-source shortest-paths problem in the general case in which edge weights may be negative. The Bellman-Ford algorithm returns a boolean value indicating whether or not there is a negative-weight cycle that is reachable from the source. If there is such a cycle, the algorithm indicates that no solution exists. If there is no such cycle, the algorithm produces the shortest paths and their weights.

The Bellman-Ford algorithm can be used for undirected graph, as a matter of fact any undirected graph is also a directed graph. You just have to specify any edges {u, v} twice (u, v) and (v, u). But don’t forget, that this also means any edge with a negative weight will count as a loop. As the Bellman-Ford algorithm ONLY works on graphs that don’t contain any cycles with negative weights this actually means your un-directed graph mustn’t contain any edges with negative weight. If it doesn’t its pretty fine to use Bellman-Ford. Since the graph has to have only nonnegative edges, this means that you might want to just use Dijkstra’s algorithm instead, since it’s asymptotically faster.

System of difference constraints

Solving systems of difference constraints

We can use the Bellman-Ford algorithm to solve a system of difference constraints. Because the constraint graph contains edges from the source vertex v0v_0v0​ to all other vertices, any negative-weight cycle in the constraint graph is reachable from v0v_0v0​. If the Bellman-Ford algorithm returns TRUE, then the shortest-path weights give a feasible solution to the system. If the Bellman-Ford algorithm returns FALSE, there is no feasible solution to the system of difference constraints.

A system of difference constraints with m constraints on n unknowns produces a graph with n+1n + 1n+1 vertices and n+mn + mn+m edges. Thus, using the Bellman-Ford algorithm, we can solve the system in O((n+1)(m+n))O((n+1)(m+n))O((n+1)(m+n)) time. We can modify the algorithm to run in O(mn)O(mn)O(mn) time, even if mmm is much smaller than nnn.

We consider the problem of ﬁnding shortest paths between all pairs of vertices in a graph. We can solve an all-pairs shortest-paths problem by running a single-source shortest-paths algorithm ∣V∣|V|∣V∣ times, once for each vertex as the source. If all edge weights are nonnegative, we can use Dijkstra’s algorithm. The binary min-heap implementation of the min-priority queue yields a running time of O(VElg⁡V)O(VE\lg V)O(VElgV). Johnson’s algorithm for sparse graphs

Johnson’s algorithm ﬁnds shortest paths between all pairs in O(V2lg⁡V+VE)O(V^2 \lg V + VE)O(V2lgV+VE) time. For sparse graphs, it is asymptotically faster than either repeated squaring of matrices or the Floyd-Warshall algorithm. Johnson’s algorithm uses as subroutines both Dijkstra’s algorithm and the Bellman-Ford algorithm.

Johnson’s algorithm uses the technique of reweighting, which works as follows.

  • If all edge weights www in a graph are nonnegative, we can ﬁnd shortest paths between all pairs of vertices by running Dijkstra’s algorithm once from each vertex.
  • If GGG has negative-weight edges but no negative-weight cycles, we simply compute a new set of nonnegative edge weights that allows us to use the same method. The new set of edge weights w^\hat{w}w^ must satisfy two important properties
    • For all pairs of vertices, a path ppp is a shortest path from uuu to vvv using weight function www if and only if ppp is also a shortest path from uuu to vvv using weight function w^\hat{w}w^.
    • For all edges, the new weight w^(u,v)\hat{w}(u,v)w^(u,v) is nonnegative.

In this section, we introduce a generic minimum-spanning-tree method that grows a spanning tree by adding one edge at a time. The generic method manages a set of edges AAA, maintaining the following loop invariant:

Prior to each iteration, AAA is a subset of some minimum spanning tree.

At each step, we determine an edge (u,v)(u,v)(u,v) that we can add to AAA without violating this invariant, in the sense that A∪{(u,v)}A\cup\{(u,v)\}A∪{(u,v)} also a subset of a minimum spanning tree. We call such an edge a safe edge for A, since we can add it safely to A while maintaining the invariant.

In the remainder of this section, we provide a rule for recognizing safe edges. The next section describes two algorithms that use this rule to ﬁnd safe edges efﬁciently.

We ﬁrst need some deﬁnitions.

cut: (S,V−S)(S,V-S)(S,V−S) is a partition of GGG.
crosses: We say that an edge (u,v)(u,v)(u,v) crosses the cut (S,V−S)(S,V-S)(S,V−S) if one of its endpoints is in SSS and the other is in V−SV-SV−S.
respect: We say that a cut respects a set AAA of edges if no edge in AAA crosses the cut.
light edge: An edge is a light edge crossing a cut if its weight is the minimum of any edge crossing the cut. Note that there can be more than one light edge crossing a cut in the case of ties.

We can prove, for every cut that respects AAA, light edge (u,v)(u,v)(u,v) is safe for A.
As the method proceeds, the set A is always acyclic; otherwise, a minimum spanning tree including A would contain a cycle, which is a contradiction.

Prim

In Prim’s algorithm, the set AAA forms a single tree. The safe edge added to AAA is always a least-weight edge connecting the tree to a vertex not in the tree.

Property: Prim’s algorithm has the property that the edges in the set AAA always form a single tree. Each step adds to the tree AAA a light edge that connects AAA to an isolated vertex - one on which no edge of AAA is incident.

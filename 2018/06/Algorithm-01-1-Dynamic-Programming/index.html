Basic Idea

Here are some examples of dynamic programming

  • rod-cut problem
  • matrix-chaining multiplication
  • longest common subsequence
  • optimal binary search tree

Here it is important to state the four steps when apply the dp algorithm.

  1. Characterize the structure of an optimal solution.
  2. Recursively deﬁne the value of an optimal solution.
  3. Compute the value of an optimal solution, typically in a bottom-up fashion.
  4. Construct an optimal solution from computed information.

Rod-cutting problem

Details

The rod-cutting problem is the following. Given a rod of length n inches and a table of prices pip_ipi​ for i=1,2,...,ni = 1,2,...,ni=1,2,...,n, determine the maximum revenue rnr_nrn​ obtainable by cutting up the rod and selling the pieces. Note that if the price pnp_npn​ for a rod of length n is large enough, an optimal solution may require no cutting at all.

Solution

We can cut up a rod of length n in 2n−12^{n-1}2n−1 different ways, since we have an independent option of cutting, or not cutting, at distance iii inches from the left end, for i=1,2,...,ni = 1,2,...,ni=1,2,...,n. If an optimal solution cuts the rod into k pieces, for some 1≤k≤n1 \leq k \leq n1≤k≤n. We denote a decomposition into pieces using ordinary additive notation, so that 7=2+2+37=2+2+37=2+2+3 indicates that a rod of length 7 is cut into three pieces—two of length 2 and one of length 3. Thus, an optimal decomposition is:

n=i1+i2+...+ikn=i_1+i_2+...+i_k n=i1​+i2​+...+ik​

of the rod into pieces of lengths i1,i2,...,iki_1, i_2,...,i_ki1​,i2​,...,ik​ provides maximum corresponding revenue

rn=pi1+pi2+...+pikr_n=p_{i_1}+p_{i_2}+...+p_{i_k} rn​=pi1​​+pi2​​+...+pik​​

We can frame the values rnr_nrn​ for n≥1n\geq1n≥1 in terms of optimal revenues from shorter rods:

rn=max⁡(pn,r1+rn−1,r2+rn−2,...,rn−1+r1)r_n=\max(p_n, r_1+r_{n-1},r_2+r_{n-2},...,r_{n-1}+r_1) rn​=max(pn​,r1​+rn−1​,r2​+rn−2​,...,rn−1​+r1​)

the maximum revenue obtained by making an initial cut of the rod into two pieces of size iii and n−in-in−i, for each i=1,2,...,ni = 1,2,...,ni=1,2,...,n, and then optimally cutting up those pieces further. Once we make the ﬁrst cut, we may consider the two pieces as independent instances of the rod-cutting problem. The overall optimal solution incorporates optimal solutions to the two related subproblems, maximizing revenue from each of those two pieces.

We can also simplify the formula, because we do not need to calculate the last half of piece. That means, only the remainder, and not the ﬁrst piece, may be further divided. We may view every decomposition of a length-n rod in this way: as a ﬁrst piece followed by some decomposition of the remainder.

rn=max⁡1≤i≤n(pi+rn−i)r_n = \max_{1\leq i \leq n}(p_i+r_{n-i}) rn​=1≤i≤nmax​(pi​+rn−i​)

In this formulation, an optimal solution embodies the solution to only one related subproblem—the remainder—rather than two.

Recursive top-down implementation

This simple program requires O(2n)O(2^n)O(2n) . Why is CUT-ROD so inefﬁcient? The problem is that CUT-ROD calls itself recursively over and over again with the same parameter values.

Dynamic programming for optimal rod-cutting

We arrange for each subproblem to be solved only once, saving its solution. If we need to refer to this subproblem’s solution again later, we can just look it up, rather than recompute it. Dynamic programming thus uses additional memory to save computation time; it serves an example of a time-memory trade-off. See this for detail information.

Top-down with memoization

Bottom-up method

For the bottom-up dynamic-programming approach, BOTTOM-UP-CUT-ROD uses the natural ordering of the subproblems: a problem of size i is “smaller” than a subproblem of size j if i<ji<ji<j. Thus, the procedure solves subproblems of sizes j=0,1,...,nj = 0,1,...,nj=0,1,...,n, in that order.

Reconstruction

We can extend the dynamic-programming approach to record not only the optimal value computed for each subproblem, but also a choice that led to the optimal value.

Matrix-chain multiplication

Details

A product of matrices is fully-parenthesized if it is either a single matrix or the product of two fully parenthesized matrix products, surrounded by parentheses. For example, if the chain of matrices is ⟨A1,A2,A3,A4⟩\langle A_1,A_2,A_3,A_4\rangle⟨A1​,A2​,A3​,A4​⟩, then we can fully parenthesize the product $A_1A_2A_3A_4 $in ﬁve distinct ways:

The cost of multiplying two matrices is pipkpjp_ip_kp_jpi​pk​pj​.

We state the matrix-chain multiplication problem as follows: given a chain ⟨A1,A2,...,An⟩\langle A_1,A_2,...,A_n\rangle⟨A1​,A2​,...,An​⟩ of n matrices, where for i=1,...,ni=1,...,ni=1,...,n, matrix AiA_iAi​ has dimension pi−1×pip_{i-1} \times p_{i}pi−1​×pi​. fully parenthesize the product A1...AnA_1...A_nA1​...An​ in a way that minimizes the number of scalar multiplications.

So how to count the number of parenthesizations?

Before solving the matrix-chain multiplication problem by dynamic programming, let us convince ourselves that exhaustively checking all possible parenthesizations does not yield an efﬁcient algorithm.

Denote the number of alternative parenthesizations of a sequence of n matrices by P(n)P(n)P(n). We summarize the costs from every break because we need to check all possible parenthesization.

P(n)={1if n=1∑k=1nP(k)P(n−k)if n≥2P(n)=\begin{cases} 1&\text{if }n=1\\ \sum_{k=1}^{n}P(k)P(n-k)&\text{if }n\geq2 \end{cases} P(n)={1∑k=1n​P(k)P(n−k)​if n=1if n≥2​

Which should be Ω(2n)\Omega(2^n)Ω(2n).

Applying dynamic programming

Step 1: The structure of an optimal parenthesization

For convenience, let us adopt the notation Ai..jA_{i..j}Ai..j​, where i≤ji\leq ji≤j, for the matrix that results from evaluating the product AiAi+1..AjA_iA_{i+1}..A_jAi​Ai+1​..Aj​ (inclusive).

The optimal substructure of this problem is as follows. Suppose that to optimally parenthesize AiAi+1...AjA_iA_{i+1}...A_jAi​Ai+1​...Aj​, we split the product between AkA_kAk​ and Ak+1A_{k+1}Ak+1​. Why? If there were a less costly way to parenthesize AiAi+1...AlA_iA_{i+1}...A_lAi​Ai+1​...Al​, then we could substitute that parenthesization in the optimal parenthesization of AiAi+1...AjA_iA_{i+1}...A_jAi​Ai+1​...Aj​ to produce another way to parenthesize AiAi+1...AjA_iA_{i+1}...A_jAi​Ai+1​...Aj​ whose cost was lower than the optimum: a contradiction.

Step 2: A recursive solution

Next, we deﬁne the cost of an optimal solution recursively in terms of the optimal solutions to subproblems. For the matrix-chain multiplication problem, we pick as our subproblems the problems of determining the minimum cost of parenthesizing AiAi+1...AjA_iA_{i+1}...A_jAi​Ai+1​...Aj​ for 1≤i≤j≤n1 \leq i \leq j \leq n1≤i≤j≤n. For reconstruct solution, we define s[i,j]s[i,j]s[i,j] to be a value of k at which we split the product Ai...AjA_i...A_jAi​...Aj​ in an optimal parenthesization.

Step 3: Computing the optimal costs

Observe that we have relatively few distinct subproblems: one subproblem for each choice of i and j satisfying 1≤i≤j≤n1 \leq i \leq j \leq n1≤i≤j≤n, or Θ(n2)\Theta(n^2)Θ(n2) in all. This property of overlapping subproblems is the second hallmark of when dynamic programming applies (the ﬁrst hallmark being optimal substructure).

Longest common subsequence

Detail

A subsequence of a given sequence is just the given sequence with zero or more elements left out. Formally, given a sequence X=\langle x_1,x_2,...,x_m\rangle​, another sequence Z=\langle z_1,z_2,...,z_k\rangle​ is a subsequence of X if there exists a strictly increasing sequence i_1,..,i_k​ of indices of X such that for all j=1,2,..,k​, we have X_{i_j}=Z_j​.
Given two sequences X and Y, we say that a sequence Z is a common subsequence of X and Y if Z is a subsequence of both X and Y.
In the longest-common-subsequence(LCS) problem, we are given two sequences X=\langle x_1,x_2,...,x_m\rangle​ and Y=\langle y_1,y_2,...,y_n\rangle​ and wish to ﬁnd a maximum-length common subsequence of X and Y .

Step 1: Characterizing a longest common subsequence

We define ith prefix of X as Xi=⟨x1,..,xi⟩X_i=\langle x_1,..,x_i \rangleXi​=⟨x1​,..,xi​⟩.

The LCS problem has an optimal-substructure property, as follow:

Let X=⟨x1,x2,...,xm⟩X=\langle x_1,x_2,...,x_m\rangleX=⟨x1​,x2​,...,xm​⟩ and Y=⟨y1,y2,...,yn⟩Y=\langle y_1,y_2,...,y_n\rangleY=⟨y1​,y2​,...,yn​⟩ be sequences, and let Z=⟨z1,z2,...,zi⟩Z=\langle z_1,z_2,...,z_i\rangleZ=⟨z1​,z2​,...,zi​⟩ be any LCS of X and Y.

  1. If xm=ynx_m = y_nxm​=yn​, then zk=xm=ynz_k = x_m = y_nzk​=xm​=yn​ and Zk−1Z_{k-1}Zk−1​ is an LCS of Xm−1X_{m-1}Xm−1​ and Yn−1Y_{n-1}Yn−1​.
  2. If xm≠ynx_m \neq y_nxm​≠yn​, then zk≠xmz_k\neq x_mzk​≠xm​ implies ZZZ is an LCS of Xm−1X_{m-1}Xm−1​ and YnY_nYn​.
  3. If xm≠ynx_m \neq y_nxm​≠yn​, then zk≠ynz_k\neq y_nzk​≠yn​ implies ZZZ is an LCS of XmX_mXm​ and Yn−1Y_{n-1}Yn−1​.

Step 2: A recursive solution

Let us deﬁne c[i,j]c[i,j]c[i,j] to be the length of an LCS of the sequences XiX_iXi​ and YjY_jYj​.

Step 3: Computing the length of an LCS

Procedure LCS-LENGTH stores the c[i,j]c[i,j]c[i,j] values in a table c[0..m,0..n]c[0..m,0..n]c[0..m,0..n] , and it computes the entries in row-major order. (That is, the procedure ﬁlls in the ﬁrst row of c from left to right, then the second row, and so on.) The procedure also maintains the table b[1..m,1..n]b[1..m,1..n]b[1..m,1..n] to help us construct an optimal solution. Intuitively, b[i,j]b[i,j]b[i,j] points to the table entry corresponding to the optimal subproblem solution chosen when computing c[i,j]c[i,j]c[i,j].

It requires O(mn)O(mn)O(mn) time and O(mn)O(mn)O(mn) space.

Step 4: Constructing an LCS

We simply begin at b[m,n]b[m,n]b[m,n] and trace through the table by following the arrows. Whenever we encounter a LEFT-UP in entry b[i,j]b[i,j]b[i,j], it implies that xi=yjx_i = y_jxi​=yj​ is an element of the LCS. With this method, we encounter the elements of this LCS in reverse order.

It requires O(m+n)O(m+n)O(m+n) time.

Improving the code

In the LCS algorithm, for example, we can eliminate the b table altogether. Each c[i,j]c[i,j]c[i,j] entry depends on only three other c table entries: c[i−1,j]c[i-1,j]c[i−1,j], c[i,j−1]c[i,j-1]c[i,j−1], c[i−1,j−1]c[i-1,j-1]c[i−1,j−1]. Given the value of c[i,j]c[i,j]c[i,j], we can determine in O(1)O(1)O(1) time which of these three values was used to compute c[i,j]c[i,j]c[i,j], without inspecting table b.

We can reduce the asymptotic space requirements for LCS-LENGTH, since it needs only two rows of table c at a time: the row being computed and the previous row. This improvement works if we need only the length of an LCS; if we need to reconstruct the elements of an LCS, the smaller table does not keep enough information to retrace our steps in O(m+n)O(m+n)O(m+n) time.

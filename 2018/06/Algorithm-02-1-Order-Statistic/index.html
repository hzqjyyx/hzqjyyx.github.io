Order Statistics

Input: A set A of n (distinct) numbers and an integer i, with 1≤i≤n1\leq i \leq n1≤i≤n.
Output: The element x∈Ax \in Ax∈A that is larger than exactly i−1i -1i−1 other elements of A.

Min-heap

We can find k’th smallest element in time complexity better than O(nLogn). A simple optomization is to create a Min Heap of the given n elements and call extractMin() k times.

Max-heap

We can also use Max Heap for finding the k’th smallest element. Following is algorithm.

  1. Build a Max-Heap MH of the first k elements (arr[0] to arr[k-1]) of the given array. O(k)

  2. For each element, after the k’th element (arr[k] to arr[n-1]), compare it with root of MH.
    ……a) If the element is less than the root then make it root and call heapify for MH
    ……b) Else ignore it.
    // The step 2 is O((n-k)*logk)

  3. Finally, root of the MH is the kth smallest element.

Time complexity of this solution is O(k + (n-k)*Logk)

Red-Black Tree - Augmented Data Structure

An order-statistic tree T is simply a red-black tree with additional information stored in each node. Besides the usual attributes, we have x.size. This attribute contains the number of (internal) nodes in the subtree rooted at x (including x itself), that is, the size of the subtree. If we deﬁne the sentinel’s size to be 0—that is, we set T.nil.size to be 0—then we have the identity:

x.size=x.left.size+x.right.size+1x.size=x.left.size+x.right.size+1 x.size=x.left.size+x.right.size+1

We do not require keys to be distinct in an order-statistic tree. We remove this ambiguity for an order-statistic tree by deﬁning the rank of an element as the position at which it would be printed in an inorder walk of the tree.

Retrieve

OS-SELECT(x,i) returns a pointer to the node containing the ith smallest key in the subtree rooted at x.

We compute r, the rank of node x within the subtree rooted at x. The value of x.left.size is the number of nodes that come before x in an inorder tree walk of the subtree rooted at x. Thus, x.left.size+1x.left.size + 1x.left.size+1 is the rank of x within the subtree rooted at x.

  • If i<ri<ri<r, then the ith smallest element resides in x’s left subtree, and so we recurse on x.left in line 5.
  • If i>ri>ri>r, then the ith smallest element resides in x’s right subtree. Since the subtree rooted at x contains r elements that come before x’s right subtree in an inorder tree walk, the ith smallest element in the subtree rooted at x is the (i−r)(i -r)(i−r)th smallest element in the subtree rooted at x.right. Line 6 determines this element recursively.

Because each recursive call goes down one level in the order-statistic tree, the total time for OS-SELECT is at worst proportional to the height of the tree. Since the tree is a red-black tree, its height is O(lg⁡n)O(\lg n)O(lgn), where n is the number of nodes. Thus, the running time of OS-SELECT is O(lg⁡n)O(\lg n)O(lgn) for a dynamic set of n elements.

Maintaining the subtree size

We shall now show how to maintain subtree sizes for both insertion and deletion without affecting the asymptotic running time of either operation.

Insertion

Insertion into a red-black tree consists of two phases.

  • The ﬁrst phase goes down the tree from the root, inserting the new node as a child of an existing node.

    • we simply increment x.size for each node x on the simple path traversed from the root down toward the leaves.
    • The new node added gets a size of 1.

  • The second phase goes up the tree, changing colors and performing rotations to maintain the red-black properties.

    • The only structural changes to the underlying red-black tree are caused by rotations, of which there are at most two.

    • Moreover, a rotation is a local operation: only two nodes have their size attributes invalidated.

Deletion

Deletion from a red-black tree also consists of two phases:

  • the ﬁrst operates on the underlying search tree. The ﬁrst phase either removes one node y from the tree or moves upward it within the tree.
    • To update the subtree sizes, we simply traverse a simple path from node y (starting from its original position within the tree) up to the root, decrementing the size attribute of each node on the path.
  • the second causes at most three rotations and otherwise performs no structural changes.
    • We handle the O(1)O(1)O(1) rotations in the second phase of deletion in the same manner as for insertion.

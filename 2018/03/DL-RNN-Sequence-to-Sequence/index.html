Text modeling

Model

Goal: Learn a model that can predict the next character given a sequence of characters, or, at a higher level, words. After observing inputs w0...wkw_0...w_kw​0​​...w​k​​, it predicts wk+1w_{k+1}w​k+1​​. Here www represents input, not weights.

Representing words

Input: one-hot vector, dimensionality of the vector is the size of the “vocabulary”
Output: probability distribution over characters, can be logloglog. Y(t,i)=P(Vi∣w0,...,wt−1)Y(t, i)=P(V_i|w_0,...,w_{t-1})Y(t,i)=P(V​i​​∣w​0​​,...,w​t−1​​). It may not be the final output, which should be a phoneme or a label, not probability distribution.
Divergence: Div(D(1...T),Y(1...T))=∑tXent(D(t),Y(t))=−∑tlogY(t,wt+1)Div(D(1...T), Y(1...T))=\sum_t Xent(D(t),Y(t))=-\sum_t logY(t, w_{t+1})Div(D(1...T),Y(1...T))=∑​t​​Xent(D(t),Y(t))=−∑​t​​logY(t,w​t+1​​).

By applying one-hot vector as input:
Good thing: makes no assumptions about the relationships between words
Bad thing: One-hot vector has low density (sparse) and will be a problem when the vocabulary is huge. Inefficient!
Solution: Projection. project the points to a lower-dimensional subspace. We can multiply a N*M matrix P which is learned. Distances in the projected space will reflect relationships imposed by the objective.

Also, P can be a MLP with linear activation.

Embeddings

It will automatically learn the relative arrangement of these symbols by throwing appropriate prediction task.

Generate Language: Synthesis

  • Draw a word from the last output’s distribution.
  • Feed the drawn word as the next word in the series, and draw the next word from the output probability distribution
  • Continue this process until we terminate generation. In some cases, e.g. generating programs, there may be a natural termination

However, it is not suitable for question answering, for answer only expected after the question ends, like Color of sky->blue. For now, we just focus on the problems where uniform weights are reasonable.

Extension

Objective: Given a sequence of inputs, asynchronously output a sequence of symbols. This is just a simple concatenation of many copies of the simple “output at the end of the input sequence” model we just saw.
Restriction: A sequence X1...XNX_1...X_NX​1​​...X​N​​ goes in, a different sequence Y1...YMY_1...Y_MY​1​​...Y​M​​ comes out, M, N can be M<N,M=N,M>NM<N, M=N, M>NM<N,M=N,M>N. No synchrony between X and Y. But input and output sequences happen in the same order.

Actually, we have no idea about when to output symbols, we are lack of information of boundary.

We need to clarify that actual output of the network is a probability for each output symbol given all inputs until that time. y4D=prob(s4=D∣X0...X4)y_4^D=prob(s_4=D|X_0...X_4)y​4​D​​=prob(s​4​​=D∣X​0​​...X​4​​). And we define that most likely symbol sequence given inputs is

Decode

GAP: Out RNN network’s output is a probability distribution among characters and what we want is actually a sequence of asynchronously non-fixed number of symbols.

We will refer to the process of obtaining an output from the network as decoding.

Here are solutions:

  • select most probable symbol at each time
  • impose external constraints
    • only allow sequences corresponding to dictionary words
    • Sub-symbol units (like in HW1)
    • Explictly arrange the constructed table

Select most probable

Simply select the most probable symbol at each time. Merge adjacent repeated symbols, and place the actual emission of the symbol in the final instant. However, there are serval problems:

  • we cannot control the number of final outputs, as well as
  • we may predict symbols that do not even appear at labels. For example, /EH/ should not be appeared at final output.
  • cannot distinguish between an extended symbol and repetitions of the symbol

Impose external constraints

Blacklist words

Only allow sequences corresponding to dictionary words, block out all rows that do not include symbolsfrom the target sequence. E.g. Block out rows that are not /B/ /IY/ or /F/.

Not a good choice. This still doesn’t assure that the decode sequence correctly expands the target symbol sequence.

Sub-symbol units

(like in HW1)

Arrange constructed table

Arrange the constructed table so that from top to bottom it has the exact sequence of symbols required. If a symbol occurs multiple times, we repeat the row in the appropriate location.

Also, constrain that the first symbol in the decode must be the top left block. The last symbol must be the bottom right. The rest of the symbols must follow a sequence that monotonically travels down from top left to bottom right, never goes up. This guarantees that the sequence is an expansion of the target sequence. /B/ /IY/ /F/ /IY/ in this case.

Whole solution:

  • Compose a graph such that every path in the graph from source to sink represents a valid alignment
    • Which maps on to the target symbol sequence /B/ /IY/ /F/ /IY/
  • Edge scores are 1
  • Node scores are the probabilities assigned to the symbols by the neural network
  • The “score” of a path is the product of the probabilities of all nodes along the path
  • Find the most probable path from source to sink using any dynamic programming algorithm
    • E.g. The Viterbi algorithm.

Alignment

If only the sequence of output symbols is provided for the training data. But no indication of which one occurs where. In other words, no timing information provided.

Guess the alignment

Concepts

  • Initialize: Assign an initial alignment
    • Either randomly, based on some heuristic, or any other rationale
  • Iterate:
    • Train the network using the current alignment
    • Reestimate the alignment for each training instance
      • Using the decoding methods already discussed

Estimating an alignment

Given:

  • The unaligned KKK-length symbol sequence S=S0...SK−1S=S_0...S_{K-1}S=S​0​​...S​K−1​​ (e.g. /B/ /IY/ /F/ /IY/)
  • An NNN-length input (N≥KN \ge KN≥K)
  • And a (trained) recurrent network

Find:

  • An NNN-length expansion s0,...sN−1s_0,...s_{N-1}s​0​​,...s​N−1​​ comprising the symbols in SSS in strict order
    • E.g. /B/ /B/ /IY/ /IY/ /IY/ /F/ /F/ /F/ /F/ /IY/...
  • si=Sk,i≤ks_i=S_k, i \le ks​i​​=S​k​​,i≤k, means SSS is a repeat version of sss
  • si=Sk,sj=Sl,i<j→k≤ls_i=S_k,s_j=S_l, i<j \to k \le ls​i​​=S​k​​,s​j​​=S​l​​,i<j→k≤l, means SSS and sss has the same order.

Outcome: an alignment (s0,...sN−1s_0,...s_{N-1}s​0​​,...s​N−1​​) of the target symbol sequence S0,...,SK−1S_0,...,S_{K-1}S​0​​,...,S​K−1​​ to the input X0...XN−1X_0...X_{N-1}X​0​​...X​N−1​​. Using dynamic programming algorithm like viterbi algorithm to compute the alignment.

Gradient

Iterative update

Option 1: For whole training set, calculate alignment at first

  • Determine alignments for every training instance
  • Train model (using SGD or your favorite approach) on the entire training set
  • Iterate

Option 2: Determine alignment during training

  • During SGD, for each training instance, find the alignment during the forward pass
  • Use in backward pass

Problem:
Approach heavily dependent on initial alignment
Prone to poor local optima
Because we try to commit our alignment assumption based on initial guess of alignment, which may be a mistake and lead to poor local optima.

Alternate solution: Do not commit to an alignment during any pass…

Appendix

Viterbi algorithm

At each node, keep track of

  • The best incoming edge
  • The score of the best path from the source to the node

Dynamically compute the best path from source to sink

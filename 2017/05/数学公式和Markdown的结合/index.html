NexT自身支持MathJax

NexT自身是支持MathJax的，默认关闭。但是开启方式十分简单。

编辑 主题配置文件， 将 mathjax 下的 enable 设定为 true 即可。 cdn 用于指定 MathJax 的脚本地址，默认是 MathJax 官方提供的 CDN 地址。

缺陷

但是刚用了20分钟，我就遇到了一个问题：写下标的时候LaTex和Markdown中的_重复了！在Typora中预览的好好的，但是hexo就先把_按照Markdown解析，由此破坏了公式，基本上这一段的公式全都出了问题。（现在已经安装完成，等以后补图）

改进

通过查阅资料，知道了问题来源是_冲突导致。
发现以下几个网站提供了多种不同解决办法，都很值得一试。

  • 如何处理Hexo和MathJax的兼容问题
  • hexo-renderer-pandoc
  • Github issues（几乎包含了全部解决办法）

解决方案

Hexo下mathjax的转义问题这篇博客已经解释的非常完全了，基本把4种方案全部说到。

推荐使用替换Markdown渲染引擎，替换成Pandoc，可以完美解决大部分问题。

  • hexo-renderer-pandoc，Pandoc和标准的Markdown语法基本一致，渲染也没有问题，推荐使用。
  • hexo-renderer-kramed，作者fork了 hexo-renderer-marked 项目，并且只针对MathJax支持进行了改进，但是还是有一些小问题，不过和hexo默认的非常接近。

替换

我选择使用Pandoc替换默认引擎。

  • 安装Pandoc
  • 进入blog目录
  • 卸载默认Markdown引擎npm uninstall hexo-renderer-marked --save(如果有安装其他的也请一并卸载)
  • 安装新引擎npm install hexo-renderer-pandoc --save

现在应该就已经可以啦！
